# v0.5.0 Release Notes

## Summary

Version `0.5.0` focuses on HSE2 wrapper-material usability, Windows DPAPI support,
and standalone experimental HSE2 GUI access. Existing HSE1 batch workflows and the
main GUI remain compatible.

## Highlights

### Windows DPAPI wrapper provider

- Added a Windows-only `dpapi` password-source provider.
- Added `dpapi-protect` for creating DPAPI-protected wrapper-material blobs.
- Supports `current_user` and `local_machine` DPAPI scopes.
- Refuses output overwrite by default unless `--force` is provided.
- Reports explicit unavailable errors on non-Windows platforms.

### HSE2 wrapper provider matrix

- Documented supported HSE2 wrapper providers:
  - `literal`
  - `env`
  - `file`
  - `keyfile`
  - `dpapi`
  - `command`
  - `prompt`
- Added smoke-tested HSE2 wrapper-provider examples.

### HSE2 DPAPI workflow coverage

- Added Windows DPAPI helper tests.
- Added `dpapi-protect` CLI tests.
- Added HSE2 encrypt/validate/decrypt workflow tests using the `dpapi` wrapper provider.

### Standalone HSE2 experimental GUI

- Added `high-security-encryptor-hse2-gui`.
- Added a reusable HSE2 experimental tab component.
- Added a standalone `HSE2 实验工具` GUI launcher.
- Added a main-GUI-facing helper for opening the standalone HSE2 window later.

### Windows EXE packaging

Windows release artifacts now include:

- `high-security-encryptor.exe`
- `high-security-encryptor-gui.exe`
- `high-security-encryptor-hse2-gui.exe`
- `README.md`
- `windows_exe.md`

## Verification

The 0.5.0 release-prep branch passed CI on Windows for:

- Python 3.11
- Python 3.12
- Python 3.13

The CI gate includes:

- editable package install;
- committed-secret scan;
- dependency vulnerability audit;
- syntax check with `compileall`;
- full unittest suite;
- console script smoke test.

## Compatibility

This release does not change existing HSE1 defaults, existing batch workflows,
existing validation behavior, existing migration behavior, existing bundle behavior,
or current main-GUI behavior.

HSE2 remains explicit and config-driven. The standalone HSE2 GUI delegates to the
existing CLI workflows instead of reimplementing cryptographic logic in the GUI layer.
